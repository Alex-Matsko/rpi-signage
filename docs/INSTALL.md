# Установка агента на устройство (Raspberry Pi / Intel NUC)

Пошаговая инструкция: как подготовить конечное устройство, установить агент,
подключить его к серверу и всё проверить. Агент одинаково работает на
Raspberry Pi 2/3/4 и на x86-мини-ПК (Intel NUC и аналогах) — отличается
только подготовка ОС.

> Предполагается, что **сервер уже развёрнут** и доступен по адресу вида
> `https://signage.example.com` (см. [../README.md](../README.md)).

---

## Шаг 1. Создайте экран на сервере и получите код подключения

1. Откройте веб-интерфейс сервера, войдите.
2. **Экраны → добавьте город** (если ещё нет) и **добавьте экран**
   (например, «Касса №1»).
3. На странице экрана появится **одноразовый код подключения** вида
   `AB12-CD34`. Он понадобится на шаге 4.

---

## Шаг 2. Подготовьте устройство

### Вариант A. Raspberry Pi 2 / 3 / 4

1. Скачайте **Raspberry Pi Imager** и запишите на microSD-карту
   **Raspberry Pi OS Lite (64-bit, Bookworm)** — рабочий стол не нужен.
2. В настройках Imager (шестерёнка) заранее задайте:
   - имя пользователя и пароль;
   - Wi-Fi (если нет провода) и страну;
   - включите **SSH** (пригодится, но не обязателен — есть веб-панель).
3. Вставьте карту, подключите HDMI к телевизору, включите питание.
4. Убедитесь, что есть интернет (провод в разъём Ethernet или заранее
   настроенный Wi-Fi).

Аппаратный декодер видео на RPi понимает **H.264 (AVC) до 1080p/30fps** —
сервер приводит контент к этому формату автоматически.

### Вариант B. Intel NUC (или другой x86-мини-ПК)

1. Установите **Debian 12** или **Ubuntu Server 24.04** (Lite/минимальная,
   без графической оболочки).
2. При установке отметьте пакеты: **standard system utilities** и
   **SSH server**. Рабочий стол не нужен — агент выводит видео напрямую
   через KMS/DRM.
3. Подключите HDMI к телевизору, обеспечьте выход в интернет.

> На NUC видео декодируется через VA-API/аппаратный блок Intel — форматы
> шире, чем на RPi, но для единообразия сервер всё равно готовит H.264.

---

## Шаг 3. (Необязательно) Проверьте вывод на телевизор

Прежде чем ставить агент, можно убедиться, что устройство вообще выводит
картинку на ТВ:

```bash
sudo apt-get update && sudo apt-get install -y mpv
# короткий тест — цветные полосы на 5 секунд
mpv --fullscreen --length=5 av://lavfi:testsrc
```

Если на телевизоре появилась тестовая картинка — вывод работает.

---

## Шаг 4. Установите агент одной командой

На устройстве выполните (подставьте адрес сервера и код с шага 1):

```bash
curl -fsSL https://signage.example.com/install.sh | sudo bash -s -- \
  --server https://signage.example.com \
  --code AB12-CD34
```

Можно сразу задать пароль локальной веб-панели:

```bash
curl -fsSL https://signage.example.com/install.sh | sudo bash -s -- \
  --server https://signage.example.com --code AB12-CD34 \
  --web-password 'мой-пароль-панели'
```

Скрипт сам:

- поставит зависимости (`mpv`, `python3`, `network-manager`, звук);
- создаст системного пользователя `signage`;
- скачает и зарегистрирует агент, получит постоянный токен;
- настроит автозапуск (`systemd`-сервис `signage-agent`);
- разрешит из панели/интерфейса перезагрузку, смену имени и Wi-Fi;
- запустит агент и **локальную веб-панель** на порту `8088`.

В конце скрипт напечатает адрес панели и, если пароль не задан, напомнит
пароль по умолчанию (`admin` / `signage`).

### Привязка без кода — через веб-панель

Код можно не указывать при установке:

```bash
curl -fsSL https://signage.example.com/install.sh | sudo bash -s -- \
  --server https://signage.example.com
```

Тогда после установки откройте панель устройства `http://<IP>:8088`,
войдите и в разделе **«Сервер»** введите адрес сервера и код подключения —
устройство привяжется без командной строки. Это удобно, когда код получают
уже на месте установки.

## Шаг 5. Проверьте, что всё работает

### 5.1 На самом устройстве

```bash
systemctl status signage-agent      # active (running)
journalctl -u signage-agent -f      # живой лог: heartbeat, скачивание афиш
```

В логе через минуту должны появиться строки о скачивании контента и
`Веб-панель устройства: http://<IP>:8088`.

### 5.2 На сервере

Откройте страницу экрана в веб-интерфейсе — в течение ~1 минуты он должен
стать **online**, показать текущую афишу, версию агента, температуру и
**ссылку на локальную панель**. Кнопкой «Сделать скриншот» можно убедиться,
что на телевизоре именно то, что нужно.

### 5.3 Локальная веб-панель устройства

Откройте в браузере (из той же сети) `http://<IP-устройства>:8088`,
войдите (`admin` / пароль). Проверьте разделы:

- **Сервер** — к какому серверу привязано устройство; здесь же можно
  привязать или сменить сервер (адрес + код подключения);
- **Сеть** — текущий IP и интерфейсы; можно подключиться к Wi-Fi;
- **Звук** — выбор аудиовыхода **HDMI** или **аналоговый 3.5 мм**;
- **Хранилище** — свободное место, список кешированных афиш, удаление
  файлов и очистка кеша;
- **Система** — часовой пояс, имя устройства, смена логина/пароля панели,
  перезапуск и перезагрузка.

IP устройства также виден на странице экрана на сервере (поле «Локальная
панель»).

## Типовые проблемы

| Симптом | Что проверить |
|---|---|
| Экран не появляется online | Интернет на устройстве; правильный `--server`; `journalctl -u signage-agent` |
| «invalid pairing code» при установке | Код уже использован — на сервере нажмите «Перевыпустить код подключения» |
| Чёрный экран на ТВ | Кабель/вход HDMI; `systemctl status signage-agent`; на RPi — включён ли нужный HDMI-порт |
| Нет звука | В панели → «Звук» выберите правильный выход (HDMI vs 3.5 мм) |
| Не открывается панель :8088 | Тот же ли вы сегмент сети; не блокирует ли порт firewall |
| Видео «дёргается» на RPi 2/3 | Тяжёлый ролик — сервер уже приводит к H.264 ≤1080p30, но RPi 2 слаб для 1080p; используйте 720p |

---

## Обновление и обслуживание

- **Агент обновляется сам**: при выходе новой версии на сервере устройство
  подтянет её в течение минуты (self-update) и перезапустится.
- **Удалённо** со страницы экрана: скриншот, веб-терминал (SSH из браузера),
  перезапуск агента, перезагрузка устройства.
- **Локально**: те же перезапуск/перезагрузка и системные настройки — в
  веб-панели `:8088`.

Полезные команды на устройстве:

```bash
systemctl restart signage-agent     # перезапустить агент
systemctl status signage-agent      # состояние
journalctl -u signage-agent -e      # последние логи
cat /var/lib/signage/settings.json  # локальные настройки (пароль хранится хешем)
```
